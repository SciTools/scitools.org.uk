iris.config¶

Provides access to Iris-specific configuration values.

The default configuration values can be overridden by creating the file iris/etc/site.cfg. If it exists, this file must conform to the format defined by ConfigParser.

iris.config.RESOURCE_DIR¶

The full path to the Iris resource directory.

iris.config.MASTER_DATA_REPOSITORY¶

Local directory where master test data exists. This option is not obligatory. The master test data is the super-set of the DATA_REPOSITORY.

iris.config.DATA_REPOSITORY¶

Local directory where test data exists. This option is obligatory for Iris unit tests. This test data is a sub-set of the MASTER_DATA_REPOSITORY, if it exists. Directory contents accessed via iris.io.select_data_path().

iris.config.SAMPLE_DATA_DIR¶

Local directory where sample data exists. Defaults to “sample_data” sub-directory of the Iris package install directory. The sample data directory supports the Iris gallery. Directory contents accessed via iris.sample_data_path().

iris.config.PALETTE_PATH¶

The full path to the Iris palette configuration directory

iris.config.RULE_LOG_DIR¶

The [optional] full path to the rule logging directory used by iris.fileformats.pp.load() and iris.fileformats.pp.save().

iris.config.RULE_LOG_IGNORE¶

The [optional] list of users to ignore when logging rules.

iris.config.IMPORT_LOGGER¶

The [optional] name of the logger to notify when first imported.

iris.config.get_dir_option(section, option, default=None)¶

Returns the directory path from the given option and section, or returns the given default value if the section/option is not present or does not represent a valid directory.

iris.config.get_option(section, option, default=None)¶

Returns the option value for the given section, or the default value if the section/option is not present.
