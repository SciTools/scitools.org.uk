iris.analysis.cartography.area_weights(cube)¶

Returns an array of area weights, with the same dimensions as the cube.

This is a 2D lat/lon area weights array, repeated over the non lat/lon dimensions.

The cube must have coordinates ‘latitide’ and ‘longitude’ with contiguous bounds.

Area weights are calculated for each lat/lon cell as:

Currently, only supports a spherical datum. Uses earth radius from the cube, if present and spherical. Defaults to iris.analysis.cartography.DEFAULT_SPHERICAL_EARTH_RADIUS.

iris.analysis.cartography.get_lat_lon_contiguous_bounded_grids(cube)¶

Return 2d lat and lon bounds.

Returns array of shape (n+1, m+1).

lats, lons = cs.get_lat_lon_bounded_grids()
iris.analysis.cartography.get_lat_lon_grids(cube)¶

Return 2d lat and lon points in the requested coordinate system.

lats, lons = get_lat_lon_grids(cube)
iris.analysis.cartography.lat_lon_range(cube, mode=None)¶

Return the lat & lon range of this Cube.

If the coordinate has both points & bounds, the mode keyword can be set to determine which should be used in the min/max calculation. (Must be one of iris.coords.POINT_MODE or iris.coords.BOUND_MODE)

iris.analysis.cartography.rotate_pole(lons, lats, pole_lon, pole_lat)¶

Given an array of lons and lats, convert to lons and lats with a rotated pole.

Note

Uses proj.4 to perform the conversion.

iris.analysis.cartography.unrotate_pole(rotated_lons, rotated_lats, pole_lon, pole_lat)¶

Given an array of lons and lats with a rotated pole, convert to unrotated lons and lats.

Note

Uses proj.4 to perform the conversion.

iris.analysis.cartography.wrap_lons(lons, base, period)¶

Returns given longitudes in the range between base and base + period.

::
wrapped_lon = wrap_lons(185, -180, 360)
