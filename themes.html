Roadmap

We have outlined the work we intend to consider planning significant product releases for Iris and Cartopy for the next two quarters, with some examples. We will also attempt to outline a very high level overview of the current thinking beyond the next six months in order to show how the next two quarters contributes to the long-term plan.

Themes we plan to address are in the next two quarters are:

    Performance
    File Formats
    Installation and setup
    Interpolation and Regridding

Themes

Performance

Three areas will be addressed with the goal of improving the ‘straight-line’ speed of Iris and Cartopy. These are file I/O, plotting and processing cubes. As a non-functional requirement, performance will be a constant theme in the roadmap, but it has its own theme here as there may be periods of sustained effort planned into certain sprints.

Data Model

Extend the Iris data model to CF version v1.6 and then v1.7. Continue to influence the development of the data model and encourage the Iris science community to contribute to its development.

Python 3

Continue to prepare Iris and Cartopy for Python 3 support and to work with the Science community on when Python 3 support is appropriate.

Documentation

Documentation reviews for Iris and Cartopy will occur both as a reaction to user feedback and requests and as planned expansion to scope and depth.

File Format Support

Currently supported file formats will continue to be developed. User requests for writing of Fieldsfiles is being assessed. Additions to the supported file formats may include BUFR, NAME, ABF, LPAR, Met Office Datapoint, and GeoTiff. A better interface for reading and writing CSV and ASCII files into cubes will also be added.

Mathematical Support

Closer coupling with established libraries such as Pandas will extend the available mathematical support. Where closer coupling is not desired or not possible then we will publish templates to aid users.

Interpolation and regridding

This theme is the focus for the next few sprints and some requirements gathering and analysis has already taken place to establish the needs of users in the Iris Google Group.

Cartopy Enhancements

    Annotation (gridlines, ticks, borders, scale bar)
    provision of vector plotting (e.g. "wind arrows" and "streamlines" style)

Iris Visualisation

Further development of specific scientific visualisations and supporting features. These include:

    A shared colourbar
    Heatmaps
    Quick visualisation tool
    Plot Custimisation

Installation and setup

Includes supporting operating system architectures such as Linux, Unix flavours and Mac OS.

    PyPi
    Homebrew
    VirtualEnv
    Ubuntu apt-get repository deployment
